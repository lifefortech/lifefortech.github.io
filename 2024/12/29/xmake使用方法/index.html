二、xmake 的常见命令

以下是一些常见的 xmake 命令，可帮助你完成从配置到编译、运行、安装等多种操作：

  1. xmake f (或 xmake config)
    配置项目，可以指定编译选项、平台/架构、外部库路径等。

    • 示例：
      xmake f –tensorrt=/path/to/tensorrt –deploy=/path/to/deploy
    • 这样就为你定义的 option(“tensorrt”) 和 option(“deploy”) 传入了实际路径。

  2. xmake
    在当前目录编译项目。若没有执行过 xmake f，则会使用上一次配置的结果（或默认配置）。
    也可以指定 -r/–rebuild 强制重新编译：
    xmake -r

  3. xmake run [target] [args…]
    运行目标可执行文件并传递参数。

    • 示例：
      xmake run detect –engine=/xxx –input=xxx.mp4
    • 如果不指定 target，就运行默认的目标。

  4. xmake clean
    清理构建文件，使得下次编译能从零开始。

  5. xmake install
    安装构建好的二进制文件到指定目录，如 /usr/local/bin 等。
    一般也可用：
    xmake install -o /desired/install/path

  6. xmake uninstall
    卸载已安装的目标文件。

  7. xmake create
    创建 xmake 项目模板。

    • 示例：
      xmake create -l c++ -t console myproject
    • 这样就生成一个简单的主函数模板和 xmake.lua。

  8. xmake -h (或 xmake –help)
    查看 xmake 的整体用法帮助。

四、总结

• xmake.lua 是 xmake 的配置文件，通过 set_project、set_version、set_modes 等命令定义基本信息，通过 option(…)、target(…) 等定义可选项和目标。
• 命令行 可以用 xmake f 来配置选项，用 xmake 编译，用 xmake run 运行，用 xmake install / uninstall 安装卸载。
• 自动化：xmake 提供了 after_build、before_build 等钩子，可以在构建前后执行自定义脚本，如复制库、预处理文件等。
• 跨平台：xmake 能自动适配 Windows、Linux、macOS 等平台，对用户较为透明。
• 依赖管理：通过 add_requires(…)+add_packages(…) 方式，可以自动下载安装第三方库（在支持的情况下）。

通过以上介绍，你可以对

xmake.lua

的编写及 xmake 的常见命令有一个深入了解，并能根据自己项目需求进行灵活定制。
