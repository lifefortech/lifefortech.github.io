闭包和迭代器

Jackey Zhou2025-05-082025-05-08

闭包 (Closures) 和迭代器 (Iterators) 是 Rust 中非常强大且富有表现力的特性，它们能够让你以函数式编程的风格编写简洁、高效且安全的代码。这两个概念经常一起使用。

闭包 (Closures)

1. 什么是闭包？

闭包是一种可以捕获其环境（即定义它的作用域中的变量）的匿名函数。你可以将闭包存储在变量中、作为参数传递给其他函数、或者从函数中返回。

与函数的区别：

  • 匿名性: 闭包通常没有显式的名称（尽管可以将其赋值给一个有名称的变量）。
  • 环境捕获: 这是闭包最核心的特性。它们可以“记住”并访问其定义时所在作用域的变量。普通函数不具备这种能力（它们只能访问全局变量或传递给它们的参数）。

为什么需要闭包？

  • 作为短小、一次性的函数传递给其他函数（例如，作为回调或传递给迭代器适配器）。
  • 在需要捕获一些局部状态的场景下创建函数式对象。
  • 简化代码，使逻辑更内聚。

2. 闭包的语法

闭包的语法简洁明了，使用一对管道符 | | 包围参数列表，然后是函数体。

第一次调用闭包时，编译器会根据上下文推断其参数和返回值的具体类型。一旦类型被推断出来，该闭包的这个实例就固定为该类型签名。

3. 环境捕获

闭包可以捕获其定义作用域中的变量。捕获方式有三种，Rust 编译器会根据闭包如何使用被捕获的变量来自动选择最不严格（最灵活）的方式：

  1. 不可变借用 (&T): 闭包以不可变的方式借用环境中的值。这种闭包可以被多次调用。对应 Fn trait。
  2. 可变借用 (&mut T): 闭包以可变的方式借用环境中的值，可以修改它们。这种闭包也可以被多次调用。对应 FnMut trait。
  3. 获取所有权 (T): 闭包获取环境中值的所有权，将其移动到闭包内部。这种闭包可能只能被调用一次（如果它消耗了捕获的值）。对应 FnOnce trait。

4. 闭包的类型与 Trait (Fn, FnMut, FnOnce)

每个闭包都有其自己独特的匿名类型。这个类型的大小和结构取决于它捕获了哪些变量。
这些匿名类型会根据其捕获方式实现一个或多个 Fn 系列 trait：

  • FnOnce:

    • 适用于所有闭包。
    • 表示闭包至少可以被调用一次。如果闭包通过值（获取所有权）捕获其环境，它可能只能被调用一次（因为它会消耗掉捕获的变量）。
    • call_once(self, args) -> Output

  • FnMut:

    • 适用于那些不移动捕获变量的所有权，但可能会修改捕获变量的闭包（即通过可变引用捕获）。
    • 可以被多次调用。
    • call_mut(&mut self, args) -> Output
    • 所有实现 FnMut 的闭包也自动实现 FnOnce。

  • Fn:

    • 适用于那些既不移动捕获变量所有权，也不修改捕获变量的闭包（即通过不可变引用捕获）。
    • 可以被多次调用，并且在调用期间不会改变其状态。
    • call(&self, args) -> Output
    • 所有实现 Fn 的闭包也自动实现 FnMut 和 FnOnce。

Rust 编译器会尽可能地为闭包推断出最通用的 Fn trait。

与函数指针 (fn) 的区别:
函数指针 (fn(A) -> B) 是一种具体的类型，它指向不捕获任何环境的函数。而闭包有其自己的匿名类型，并且可以捕获环境。如果一个闭包不捕获任何环境，它可以被强制转换（coerced）为一个函数指针。

5. 将闭包传递给函数

当函数期望接收一个闭包作为参数时，通常会使用泛型和 Fn 系列 trait 来指定参数类型。

迭代器 (Iterators)

1. 什么是迭代器？

迭代器是一种允许你按顺序访问一个序列中各个元素的对象。在 Rust 中，迭代器是惰性 (lazy) 的，这意味着它们在被实际消耗之前不会执行任何操作。

为什么迭代器重要？

  • 抽象循环: 它们提供了一种比手动编写 for 或 while 循环更高级、更抽象的方式来处理序列。
  • 高效性: 由于惰性求值和编译器的优化（如循环展开和内联），迭代器通常能提供与手写循环相当甚至更好的性能（零成本抽象）。
  • 组合性: 迭代器可以被链接起来形成复杂的数据处理流水线。
  • 安全性: 减少了手动管理索引可能导致的越界错误。

3. 获取迭代器的三种方式

对于集合类型（如 Vec<T>, HashMap<K, V> 等），通常有三种方式获取迭代器：

  1. into_iter():

    • 获取集合的所有权并返回一个迭代器，该迭代器会产生集合中元素的值 (可能会移动所有权)。
    • 集合本身在此之后通常不能再被使用。
    • for item in my_collection { ... } 隐式调用 my_collection.into_iter()。

  2. iter():

    • 获取集合的不可变引用并返回一个迭代器，该迭代器会产生集合中元素的**不可变引用 (&Item)**。
    • 原始集合在此之后仍然可用。

  3. iter_mut():

    • 获取集合的可变引用并返回一个迭代器，该迭代器会产生集合中元素的**可变引用 (&mut Item)**。
    • 允许你在迭代过程中修改集合的元素。
    • 原始集合在此之后仍然可用。

4. 迭代器适配器 (Iterator Adapters)

迭代器适配器是 Iterator trait 上的方法，它们会消耗一个迭代器并产生一个新的、经过某种方式转换或过滤的迭代器。适配器是惰性的，它们本身不执行任何工作，直到最终有一个消费适配器被调用。

一些常用的迭代器适配器：

  • map<B, F>(self, f: F) -> Map<Self, F>:

    • 接收一个闭包 f，对迭代器中的每个元素应用该闭包，并返回一个包含新元素的新迭代器。
    • F 是一个 FnMut(Self::Item) -> B 类型的闭包。

  • filter<P>(self, predicate: P) -> Filter<Self, P>:

    • 接收一个闭包 predicate，该闭包对每个元素返回 bool。只保留使闭包返回 true 的元素。
    • P 是一个 FnMut(&Self::Item) -> bool 类型的闭包 (注意是对引用的判断)。

  • zip<U>(self, other: U) -> Zip<Self, U::IntoIter>: 将两个迭代器合并成一个产生元组的迭代器。当其中一个迭代器耗尽时停止。

  • rev(self) -> Rev<Self>: 反转迭代器的顺序 (需要迭代器实现 DoubleEndedIterator)。

  • skip(n: usize): 跳过迭代器前 n 个元素。

  • take(n: usize): 获取迭代器前 n 个元素。

  • enumerate(self) -> Enumerate<Self>: 将迭代器转换为产生 (index, value) 元组的迭代器。

  • filter_map<B, F>(self, f: F) -> FilterMap<Self, F>: filter 和 map 的组合。闭包返回 Option<B>，只有 Some(value) 会被保留并解包。

  • flat_map<U, F>(self, f: F) -> FlatMap<Self, U, F>: 对每个元素应用一个返回迭代器的闭包，然后将所有这些内部迭代器扁平化为一个单一的迭代器。

  • chain<U>(self, other: U) -> Chain<Self, U::IntoIter>: 连接两个迭代器。

5. 消费适配器 (Consuming Adapters / Consumers)

消费适配器（或称消费者）是 Iterator trait 上的方法，它们会驱动迭代器的执行，消耗其元素，并产生某种最终结果。

一些常用的消费适配器：

  • collect<B: FromIterator<Self::Item>>(self) -> B:

    • 最常用的消费者之一。将迭代器中的所有元素收集到一个新的集合中，例如 Vec<T>, HashMap<K, V>, String 等。
    • 目标集合类型 B 必须实现 FromIterator<Self::Item> trait。

  • sum<S: Sum<Self::Item>>(self) -> S: 计算迭代器中所有元素的总和 (元素类型必须实现 std::iter::Sum)。

  • product<P: Product<Self::Item>>(self) -> P: 计算迭代器中所有元素的乘积 (元素类型必须实现 std::iter::Product)。

  • count(self) -> usize: 返回迭代器中元素的数量。

  • last(self) -> Option<Self::Item>: 返回迭代器的最后一个元素。

  • nth(n: usize) -> Option<Self::Item>: 返回迭代器的第 n 个元素（从0开始计数），并消耗掉前面的所有元素。

  • fold<B, F>(self, init: B, f: F) -> B:

    • 一个非常强大的消费者。它接收一个初始值 init 和一个闭包 f。
    • 闭包 f 接收两个参数：一个累加器 (accumulator) 和迭代器中的一个元素，并返回新的累加器值。
    • fold 会对迭代器中的每个元素应用这个闭包，最终返回累加器的值。

  • for_each<F>(self, f: F): 对迭代器中的每个元素执行闭包 f。类似于 for 循环，但不返回值。

  • any<F>(self, f: F) -> bool: 如果迭代器中至少有一个元素使闭包 f 返回 true，则返回 true。

  • all<F>(self, f: F) -> bool: 如果迭代器中所有元素都使闭包 f 返回 true，则返回 true。

  • find<P>(&mut self, predicate: P) -> Option<Self::Item>: 返回第一个使闭包 predicate 返回 true 的元素。

6. 创建自定义迭代器

你可以通过为一个 struct 或 enum 实现 Iterator trait 来创建自己的迭代器。关键在于实现 next() 方法。

7. 迭代器与性能

Rust 的迭代器设计（特别是惰性求值和单相化）使得它们在性能上通常与手写的循环相当，有时甚至更好，因为编译器有更多的机会进行优化。你通常不必为了性能而牺牲迭代器的可读性和表达力。

闭包和迭代器是 Rust 中非常强大的组合。它们共同提供了一种优雅、高效且安全的方式来处理数据和执行重复操作。当你开始更多地使用 Rust 时，你会发现它们无处不在，尤其是在标准库和许多第三方库中。
