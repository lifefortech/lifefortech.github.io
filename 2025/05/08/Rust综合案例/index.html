Jackey Zhou2025-05-082025-05-08

由于一个例子很难完美地覆盖所有知识点而不显得过于臃生，我将提供一个相对完整的主案例，并可能在后续提供其他侧重不同方面的案例。

案例 1：简单的命令行任务管理工具

这个例子将实现一个基础的命令行任务管理工具，允许用户添加任务、查看任务、标记任务完成、移除任务，以及将任务列表保存到文件和从文件加载。

涉及的主要知识点:

  • struct 和 enum 定义数据结构
  • Vec 管理任务列表
  • 所有权和借用
  • Option 和 Result 进行错误处理
  • match 表达式处理用户输入
  • impl 和方法
  • trait (如 Display) 的实现
  • 模块化 (mod, use)
  • 文件 I/O (std::fs, std::io)
  • 错误处理 (? 运算符, 自定义错误类型)
  • 序列化/反序列化 (使用 serde 和 serde_json)
  • 日期和时间 (使用 chrono)
  • 单元测试 (#[test])
  • 迭代器和闭包

项目结构 (概念性):

首先，我们需要在 Cargo.toml 中添加依赖：

src/task.rs (任务模块)

src/main.rs (主程序逻辑)

如何编译和运行:

  1. 确保你已经安装了 Rust 和 Cargo。
  2. 将上述两个代码块分别保存到 src/task.rs 和 src/main.rs。
  3. 在 Cargo.toml 中添加之前提到的依赖。
  4. 在项目根目录下运行 cargo build 来编译。
  5. 运行 cargo run 来启动程序。
  6. 运行 cargo test 来执行单元测试。

这个例子展示了如何将多个 Rust 的核心概念结合起来构建一个有实际功能的小型应用程序。它包含了：

  • 数据建模: 使用 struct 和 enum。
  • 集合操作: 使用 Vec 存储任务，并通过迭代器进行操作。
  • 所有权与借用: 函数参数和方法接收者 (&self, &mut self) 的正确使用。
  • 错误处理: 定义自定义错误类型 TaskError，使用 Result 和 ? 运算符。
  • 模块化: 将任务相关的逻辑分离到 task.rs 模块中。
  • 文件 I/O: 使用 std::fs 和 std::io。
  • 序列化/反序列化: 使用 serde 和 serde_json 将任务列表保存为 JSON 文件。
  • 第三方库: chrono 用于时间戳，thiserror 用于简化错误类型定义。
  • 用户交互: 通过命令行进行基本的输入输出。
  • 测试: 在 task.rs 中编写单元测试。

这只是一个起点，你可以基于此进行扩展，例如：

  • 添加更复杂的任务属性（优先级、截止日期、标签等）。
  • 实现更高级的过滤和排序功能。
  • 使用更健壮的命令行参数解析库 (如 clap)。
  • 将其改为一个简单的 Web 服务 (使用 actix-web 或 axum)。
