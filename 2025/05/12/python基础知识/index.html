1. Python 简介

什么是 Python?

Python 是一种解释型、交互式、面向对象的高级编程语言。它由 Guido van Rossum 于 1989 年底发明，第一个公开发行版发行于 1991 年。Python 的设计哲学强调代码的可读性和简洁的语法（尤其是使用空格缩进来区分代码块，而不是使用大括号或关键字）。

Python 的特点

  • 易于学习与使用: 语法清晰，接近自然语言，上手快。
  • 可读性强: 代码结构清晰，强制缩进使其风格统一。
  • 动态类型: 变量类型在运行时确定，无需预先声明。
  • 解释型语言: 代码逐行执行，无需编译成机器码后再运行（实际上是编译成字节码再由 Python 虚拟机执行）。
  • 高级语言: 开发者无需关心底层细节，如内存管理（有自动垃圾回收）。
  • 跨平台: 可在 Windows, macOS, Linux/Unix, 以及其他操作系统上运行。
  • 面向对象: 支持面向对象的编程范式，也支持过程式和函数式编程。
  • 庞大的标准库: “Batteries included”，提供了大量可直接使用的模块，覆盖网络、文件、GUI、数据库、文本处理等。
  • 丰富的第三方库: 如 NumPy, Pandas (数据分析), Django, Flask (Web 开发), TensorFlow, PyTorch (机器学习) 等。
  • 可扩展性与可嵌入性: 可以用 C/C++ 编写性能关键模块，也可以将 Python 嵌入到 C/C++ 程序中。
  • 开源: 拥有活跃的社区支持。

Python 版本

  • Python 2: 曾经广泛使用，但已于 2020 年 1 月 1 日停止官方支持 (EOL)。新项目不应使用。
  • Python 3: 当前的主流版本，不断发展中。Python 3 不完全向后兼容 Python 2。本报告基于 Python 3 (特别是 3.8 及以后版本，以涵盖 f-string、字典顺序保证等现代特性)。

2. 环境搭建与基础

安装 Python

访问 Python 官方网站 (python.org) 下载适合你操作系统的最新稳定版 Python 3 安装程序。安装时，建议勾选 “Add Python to PATH” (Windows) 或确保 Python 的可执行文件路径已添加到系统环境变量中。

Python 解释器与执行脚本

  • 交互式解释器: 在命令行输入 python (或 python3，取决于你的安装和系统配置) 启动。可以直接输入 Python 代码并立即看到结果。

PEP 8 代码风格指南

PEP 8 (Python Enhancement Proposal 8) 是 Python 官方的代码风格指南。遵循 PEP 8 可以让代码更易读、易维护，并保持 Python 社区代码风格的一致性。主要建议包括：

  • 使用 4 个空格进行缩进。
  • 行长度不超过 79 个字符。
  • 类名使用驼峰命名法 (CapWords)。
  • 函数名和变量名使用小写字母和下划线 (snake_case)。
  • 常量名使用全大写字母和下划线。
  • 在运算符两侧和逗号后使用空格。

基本语法元素

  • 注释:

    • 单行注释: 以 # 开始。
    • 多行注释: 严格来说 Python 没有块注释，但可以用多个 #，或者使用未赋值的多行字符串 (三引号) 作为变通方案（解释器会忽略它们，但它们不是真正的注释，会被加载为字符串对象）。

  • 缩进: Python 使用缩进来定义代码块的层级（如函数体、循环体、条件语句体）。缩进错误会导致 IndentationError。

  • 语句: 通常一行一个语句。如果一个语句太长，可以使用反斜杠 \ 进行换行，或者在括号、方括号、花括号内的换行是自然的。

  • 标识符与关键字:

    • 标识符 (变量名、函数名、类名等) 只能包含字母、数字和下划线，且不能以数字开头。区分大小写。
    • 关键字是 Python 中有特殊含义的保留字，不能用作标识符 (如 if, for, def, class 等)。可以通过 import keyword; print(keyword.kwlist) 查看所有关键字。

3. 变量与基本数据类型

变量赋值

Python 是动态类型语言，变量在赋值时创建，无需预先声明类型。

数字 (Numbers)

  • 整数 (int): 任意大小的整数 (Python 3 中没有长整型 long 的区分)。
  • 浮点数 (float): 带小数点的数字，或使用科学计数法表示的数字。
  • 复数 (complex): 由实部和虚部组成 (如 3+5j)。较少在通用编程中使用。

字符串 (Strings)

  • 表示文本数据，不可变序列。
  • 可以用单引号 ('...')、双引号 ("...") 或三引号 ('''...''' 或 """..."""，用于多行字符串) 包围。
  • 常用操作:
    • 连接: +
    • 重复: *
    • 索引: s[i] (从 0 开始)
    • 切片: s[start:end:step]
    • 长度: len(s)
    • 成员检查: char in s
    • 格式化:
      • 旧式 % 操作符 (不推荐)
      • str.format() 方法
      • f-字符串 (Formatted String Literals, Python 3.6+，推荐)
    • 常用方法: upper(), lower(), strip(), split(), join(), find(), replace() 等。

布尔值 (Booleans)

  • 只有两个值: True 和 False (首字母大写)。
  • 通常是比较运算和逻辑运算的结果。
  • 在数字上下文中，True 等同于 1，False 等同于 0。

None 类型

  • None 是一个特殊常量，表示空值或没有值。它有自己的类型 NoneType。
  • 函数如果没有显式 return 语句，默认返回 None。

类型检查与转换

  • 类型检查: type() 函数返回对象的类型。isinstance(object, classinfo) 检查对象是否是指定类或其子类的实例。
  • 类型转换 (Coercion):
    • int(value): 转换为整数。
    • float(value): 转换为浮点数。
    • str(value): 转换为字符串。
    • bool(value): 转换为布尔值。任何非零数字、非空字符串/容器都为 True，0、None、空字符串/容器为 False。

4. 运算符

算术运算符

+ (加), - (减), * (乘), / (浮点除法), // (整数除法/地板除), % (取模/取余), ** (幂运算)

比较运算符

== (等于), != (不等于), > (大于), < (小于), >= (大于等于), <= (小于等于)。返回布尔值。

逻辑运算符

and (逻辑与), or (逻辑或), not (逻辑非)。

  • 短路求值:
    • x and y: 如果 x 为假，则返回 x 的值，否则返回 y 的值。
    • x or y: 如果 x 为真，则返回 x 的值，否则返回 y 的值。

赋值运算符

= (赋值), +=, -=, *=, /=, //=, %=, **=

成员运算符 (in, not in)

检查一个值是否存在于序列（如字符串、列表、元组）或集合中。

身份运算符 (is, is not)

检查两个操作数是否引用内存中的同一个对象。与 == 不同，== 检查值是否相等。

位运算符 (较少直接使用)

& (按位与), | (按位或), ^ (按位异或), ~ (按位取反), << (左移), >> (右移)。主要用于底层编程或特定算法。

运算符优先级

Python 有明确的运算符优先级规则 (例如 ** 高于 *//，*// 高于 +/-)。使用括号 () 可以明确运算顺序。

5. 常用内置数据结构 (Collections)

列表 (Lists)

  • 定义: 有序、可变的元素集合。用方括号 [] 定义，元素用逗号分隔。
  • 特点:
    • 可以包含不同类型的元素。
    • 通过索引访问元素 (从 0 开始)。
    • 支持切片。
    • 可变 (可以添加、删除、修改元素)。
  • 常用方法:
    • append(item): 在列表末尾添加元素。
    • insert(index, item): 在指定索引处插入元素。
    • remove(item): 移除第一个匹配的元素。
    • pop(index=-1): 移除并返回指定索引处的元素 (默认最后一个)。
    • extend(iterable): 将可迭代对象中的所有元素添加到列表末尾。
    • sort(key=None, reverse=False): 原地排序。
    • reverse(): 原地反转列表。
    • index(item): 返回第一个匹配元素的索引。
    • count(item): 返回元素出现的次数。
    • clear(): 清空列表。
    • copy(): 返回列表的浅拷贝。
  • 列表作为栈和队列:
    • 栈 (LIFO): append() 入栈, pop() 出栈。
    • 队列 (FIFO): append() 入队, pop(0) 出队 (效率不高)。更推荐使用 collections.deque 实现高效队列。

元组 (Tuples)

  • 定义: 有序、不可变的元素集合。用圆括号 () 定义，元素用逗号分隔。
  • 特点:
    • 可以包含不同类型的元素。
    • 通过索引访问元素。
    • 支持切片。
    • 不可变 (一旦创建，元素不能被修改、添加或删除)。
    • 通常用于存储不应改变的相关数据集合，或作为字典的键 (因为不可变)。

字典 (Dictionaries)

  • 定义: 无序 (Python 3.6 及更早版本) 或有序 (Python 3.7+ 版本) 的键值对 (key: value) 集合。用花括号 {} 定义。
  • 特点:
    • 键 (key) 必须是唯一的且不可变类型 (如字符串、数字、元组)。
    • 值 (value) 可以是任意类型，且可以重复。
    • 通过键访问值，而不是索引。
    • 可变。
  • 常用方法:
    • get(key, default=None): 获取键对应的值，如果键不存在则返回默认值 (不会抛出 KeyError)。
    • keys(): 返回包含所有键的视图对象。
    • values(): 返回包含所有值的视图对象。
    • items(): 返回包含所有 (键, 值) 元组的视图对象。
    • pop(key, default=None): 移除并返回键对应的值。
    • popitem(): 移除并返回一个 (键, 值) 对 (Python 3.7+ 移除最后一个，之前版本随机)。
    • update(other_dict): 用另一个字典的键值对更新当前字典。
    • clear(): 清空字典。
    • copy(): 返回字典的浅拷贝。

集合 (Sets)

  • 定义: 无序 (内部实现可能有序，但不保证)、不重复元素的集合。用花括号 {} 定义，或使用 set() 函数。创建空集合必须用 set()，因为 {} 创建的是空字典。
  • 特点:
    • 元素唯一。
    • 主要用于成员测试和消除重复元素。
    • 支持数学集合运算 (并集、交集、差集、对称差集)。
    • 可变 (可以添加、删除元素)。
  • 常用方法与操作:
    • add(item): 添加元素。
    • remove(item): 移除元素 (如果元素不存在会抛出 KeyError)。
    • discard(item): 移除元素 (如果元素不存在则什么也不做)。
    • pop(): 移除并返回任意一个元素 (集合无序)。
    • clear(): 清空集合。
    • copy(): 返回集合的浅拷贝。
    • 集合运算:
      • 并集: s1 | s2 或 s1.union(s2)
      • 交集: s1 & s2 或 s1.intersection(s2)
      • 差集: s1 - s2 或 s1.difference(s2)
      • 对称差集: s1 ^ s2 或 s1.symmetric_difference(s2)
      • 子集检查: s1 <= s2 或 s1.issubset(s2)
      • 超集检查: s1 >= s2 或 s1.issuperset(s2)
  • 不可变集合 (frozenset): frozenset 是不可变的集合，因此可以作为字典的键或集合的元素。

字符串 (作为序列的深入探讨)

字符串是字符的不可变序列。这意味着它们支持所有通用的序列操作：

  • 索引: my_string[i]
  • 切片: my_string[start:stop:step]
  • 长度: len(my_string)
  • 迭代: for char in my_string:
  • 成员检查: "a" in my_string
  • 连接: str1 + str2
  • 重复: str1 * 3
    由于其不可变性，对字符串的所有修改操作（如 replace, upper）都会返回一个新的字符串，而不是修改原始字符串。

6. 控制流

条件语句 (if, elif, else)

根据条件执行不同的代码块。

循环语句

for 循环

用于遍历可迭代对象 (如列表、元组、字符串、字典的键、集合、range() 对象等) 中的元素。

while 循环

当给定条件为真时，重复执行代码块。

循环控制 (break, continue, pass)

  • break:立即终止当前最内层循环。

  • continue: 跳过当前循环中余下的语句，并开始下一次迭代。

  • pass: 空操作语句，用作占位符，表示什么也不做。当语法上需要一个语句但逻辑上不需要任何操作时使用。

循环中的 else 子句

  • for 循环和 while 循环都可以有 else 子句。

  • 仅当循环正常完成（即没有被 break 语句中断）时，else 子句才会执行。

range() 函数

生成一个整数序列，常用于 for 循环中。

  • range(stop): 生成从 0 到 stop-1 的整数。
  • range(start, stop): 生成从 start 到 stop-1 的整数。
  • range(start, stop, step): 生成从 start 到 stop-1，步长为 step 的整数。

7. 函数 (Functions)

函数是组织好的、可重复使用的、用来执行单一或相关联功能的代码块。

定义函数 (def)

参数与参数传递

Python 中的参数传递是“赋值传递”（pass-by-assignment）或“对象引用传递”（pass-by-object-reference）。

  • 如果传递的是不可变对象（如数字、字符串、元组），函数内部对参数的重新赋值不会影响外部变量。
  • 如果传递的是可变对象（如列表、字典），函数内部对该对象的修改（例如添加元素到列表）会影响外部对象。但如果函数内部将参数重新赋值给一个全新的对象，则外部对象不受影响。

位置参数

调用函数时，参数按位置顺序匹配。

关键字参数

调用函数时，通过 参数名=值 的形式指定参数，可以不按顺序。

默认参数值

定义函数时可以为参数指定默认值。如果调用时不提供该参数，则使用默认值。
重要: 默认参数值只在函数定义时计算一次。如果默认值是可变对象（如列表或字典），多次调用函数且不提供该参数时，它们会共享同一个可变对象。

可变参数 (*args, **kwargs)

  • *args (非关键字可变参数): 将多余的位置参数收集为一个元组。
  • **kwargs (关键字可变参数): 将多余的关键字参数收集为一个字典。

仅位置参数与仅关键字参数 (Python 3.8+)

  • / (斜杠): 之前的参数只能按位置传递。
  • * (星号): 之后的参数只能按关键字传递。

返回值 (return)

函数使用 return 语句返回值。如果函数没有 return 语句，或者 return 语句后没有表达式，则默认返回 None。函数可以返回多个值，这些值会以元组的形式返回。

作用域 (LEGB 规则)

Python 查找变量的顺序：

  1. L (Local): 函数内部定义的局部作用域。
  2. E (Enclosing function locals): 嵌套函数（闭包）的外部函数作用域。
  3. G (Global): 模块级别的全局作用域。
  4. B (Built-in): Python 内置的名称 (如 print, len)。
  • global 关键字: 在函数内部修改全局变量。
  • nonlocal 关键字: 在嵌套函数内部修改外层 (非全局) 函数的变量。

匿名函数 (lambda)

一种小型的、一次性的、匿名的函数，使用 lambda 关键字定义。通常用于需要一个简单函数作为参数的场合 (如 map, filter, sorted 的 key 参数)。
语法: lambda arguments: expression

文档字符串 (Docstrings)

函数（以及类、模块）定义后的第一个语句如果是字符串字面量，则成为该对象的文档字符串。通常用三引号包围，用于解释对象的功能、参数、返回值等。可以通过 help(function_name) 或 function_name.__doc__ 查看。

递归函数

函数在其定义中调用自身。必须有一个或多个基本情况 (终止条件) 以防止无限递归。

高阶函数 (Map, Filter, Reduce)

  • 高阶函数: 接受一个或多个函数作为参数，或返回一个函数的函数。
  • map(function, iterable, ...): 将函数应用于可迭代对象的每个元素，并返回一个迭代器 (包含结果)。
  • filter(function, iterable): 用函数测试可迭代对象的每个元素，返回一个迭代器，其中包含函数返回真值的元素。
  • functools.reduce(function, iterable[, initializer]): (需要 from functools import reduce) 对可迭代对象中的项进行累积操作。 列表推导式和生成器表达式通常是 map 和 filter 更 Pythonic 的替代方案。

8. 模块与包 (Modules and Packages)

将相关的代码组织到模块中，并将相关的模块组织到包中，以提高代码的可维护性和可重用性。

模块的概念与导入 (import, from...import)

  • 模块 (Module): 一个包含 Python 定义和语句的 .py 文件。文件名就是模块名 (不带 .py 后缀)。
  • 导入模块:
    • import module_name: 导入整个模块。通过 module_name.attribute_name 访问其内容。
    • import module_name as alias_name: 导入模块并给它一个别名。
    • from module_name import name1, name2 as alias2: 从模块中导入指定的名称 (函数、类、变量)。可以直接使用 name1。
    • from module_name import *: 导入模块中的所有公共名称 (不推荐，可能导致命名空间污染)。

创建自定义模块

只需创建一个 .py 文件，其中包含函数、类和变量定义即可。

包的概念与结构

  • 包 (Package): 一个包含多个模块的文件夹 (目录)。包可以嵌套。
  • 一个目录要成为 Python 包，它必须包含一个 (通常为空的) __init__.py 文件。这个文件告诉 Python 该目录应被视为一个包。

__init__.py 文件

  • 在导入包时，__init__.py 文件会被首先执行。
  • 它可以为空，也可以包含包级别的初始化代码，或者定义 __all__ 变量来控制 from package import * 时导入哪些模块。

9. 面向对象编程 (OOP)

Python 是一门完全支持面向对象编程的语言。OOP 是一种编程范式，它使用“对象”来设计软件。对象是数据（属性）和操作数据的代码（方法）的集合。

类 (class) 与对象 (实例)

  • 类 (Class): 创建对象的蓝图或模板。它定义了一组属性和方法，这些属性和方法是该类的所有对象共有的。
  • 对象 (Object/Instance): 类的具体实例化。每个对象都有其自己的属性值，但共享类中定义的方法。

构造函数 (__init__)

  • __init__ 方法是一个特殊的方法 (魔术方法)，在创建类的新实例时自动调用。
  • 它的第一个参数通常是 self，代表正在创建的实例本身。self 不是关键字，但按惯例使用。
  • 用于初始化实例的属性。

实例属性与类属性

  • 实例属性: 在 __init__ 方法中通过 self.attribute_name = value 定义，属于特定实例。每个实例的实例属性值可以不同。
  • 类属性: 在类定义内部、任何方法之外定义的属性。由该类的所有实例共享。如果一个实例修改了它继承的类属性（实际上是创建了一个同名的实例属性覆盖了类属性），则该修改只对该实例有效；其他实例仍然访问原始的类属性。

方法 (实例方法, 类方法, 静态方法)

  • 实例方法: 类中最常见的方法类型。第一个参数总是 self，引用调用该方法的实例。可以访问和修改实例属性和类属性。

  • 类方法 (@classmethod 装饰器): 第一个参数是类本身，通常命名为 cls。可以访问和修改类属性，但不能直接访问实例属性（因为它不知道是哪个实例）。常用于工厂方法（创建类的实例）。

  • 静态方法 (@staticmethod 装饰器): 不接受隐式的第一个参数（既不是 self 也不是 cls）。它就像一个定义在类命名空间内的普通函数。它不能访问类属性或实例属性（除非显式传递）。通常用于工具函数，这些函数在逻辑上与类相关，但不需要访问类或实例的状态。

封装 (Encapsulation)

将数据（属性）和操作数据的方法捆绑在一个单元（类）中，并对外部隐藏对象的内部状态（通过访问控制）。Python 没有像 C++ 或 Java 那样的严格 private 或 protected 关键字。

  • 约定:
    • 以下划线 _ 开头的属性或方法 (如 _internal_var) 被视为“内部使用”或“受保护的”，提示外部不应直接访问，但 Python 解释器不会阻止。
    • 以双下划线 __ 开头的属性或方法 (如 __private_var) 会被 Python 进行“名称改写 (name mangling)” (变为 _ClassName__private_var)，使其更难从外部直接访问，但仍非真正的私有。这主要用于避免子类意外覆盖父类中的同名“私有”成员。

继承 (Inheritance)

允许一个类（子类/派生类）继承另一个类（父类/基类）的属性和方法。子类可以重用父类的代码，并可以添加自己的特定功能或覆盖父类的功能。

单继承与多继承

  • 单继承: 一个子类只继承自一个父类。
  • 多继承: 一个子类可以继承自多个父类。语法: class SubClass(Parent1, Parent2, ...):
    • 方法解析顺序 (MRO - Method Resolution Order): Python 使用 C3 线性化算法来确定在多继承中查找属性和方法的顺序。可以通过 ClassName.mro() 或 ClassName.__mro__ 查看。
    • 多继承可能导致复杂性（如“菱形问题”），应谨慎使用。组合通常优于继承。

方法重写 (Overriding)

子类可以定义与父类同名的方法，从而覆盖父类的行为。

super()

super() 函数用于调用父类的方法。这在子类的 __init__ 中初始化父类部分，或在子类重写的方法中扩展父类行为时非常有用。

多态 (Polymorphism) 与鸭子类型 (Duck Typing)

  • 多态: 不同类的对象可以对相同的方法调用做出不同的响应。在上面的 Animal, Dog, Cat 示例中，speak() 方法就是多态的。

  • 鸭子类型 (Duck Typing): “如果它走起来像鸭子，叫起来也像鸭子，那么它就是一只鸭子。” Python 不关心对象的类型是什么，只关心它是否具有期望的方法或属性。如果一个对象实现了某个方法，那么它就可以被用于期望该方法的上下文中，无论它是什么类型。

特殊方法 (魔术方法如 __str__, __repr__, __len__)

以双下划线开头和结尾的方法，Python 会在特定情况下自动调用它们，以实现某些内置行为或操作符重载。

  • __init__(self, ...): 构造函数。
  • __str__(self): 当对对象使用 str() 函数或 print() 时调用。应返回一个用户友好的字符串表示。
  • __repr__(self): 当对对象使用 repr() 函数或在解释器中直接输入对象名时调用。应返回一个明确的、能表示如何创建该对象的字符串表示 (如果可能)。如果没有 __str__，print() 会尝试使用 __repr__。
  • __len__(self): 当对对象使用 len() 函数时调用。应返回对象的“长度”。
  • __eq__(self, other): 定义 == 操作符的行为。
  • __add__(self, other): 定义 + 操作符的行为。
  • 还有很多，如 __getitem__ (用于索引 []), __call__ (使实例可调用 ()) 等。

属性装饰器 (@property)

允许将类的方法用作属性一样访问 (即不带括号调用)。常用于实现 getter、setter 和 deleter。

10. 错误与异常处理

程序运行时发生的错误称为异常。Python 使用异常对象来表示异常状态。

语法错误 (Syntax Errors) vs 异常 (Exceptions)

  • 语法错误 (Syntax Errors / Parsing Errors): Python 代码不符合语法规则，解释器在解析代码时就会发现并报告。程序根本无法开始运行。
  • 异常 (Exceptions): 代码语法正确，但在运行时发生错误 (如除以零、访问不存在的键、类型不匹配)。

try...except 语句

用于捕获和处理运行时可能发生的异常。

处理多种异常

可以将多个异常类型放在一个 except 子句的元组中。

else 子句 (无异常时执行)

如果 try 块中没有发生任何异常，则执行 else 子句中的代码。

finally 子句 (总是执行)

无论 try 块中是否发生异常，finally 子句中的代码总是会被执行。通常用于释放资源 (如关闭文件、网络连接)。

使用 with 语句管理资源通常更简洁。

抛出异常 (raise)

可以使用 raise 语句主动抛出一个异常。可以抛出内置异常或自定义异常。

自定义异常

通过创建继承自 Exception (或其子类) 的新类来定义自己的异常类型。

断言 (assert)

assert expression [, message]
用于在代码中插入调试检查。如果 expression 为假，则抛出 AssertionError，并可选择附带 message。
assert 主要用于开发和测试阶段，不应用于处理预期的运行时错误（应使用 try...except）。在生产环境中，可以通过 -O 选项禁用 assert 语句以提高性能。

11. 文件操作 (File I/O)

与文件系统中的文件进行交互。

打开与关闭文件 (open(), close())

  • open(file, mode='r', encoding=None): 打开文件并返回一个文件对象。
    • file: 文件路径字符串。
    • mode: 文件打开模式 (见下文)。
    • encoding: 文件的编码格式 (如 'utf-8', 'latin-1')，处理文本文件时通常需要指定。
  • file_object.close(): 关闭文件，释放资源。非常重要，否则可能导致数据丢失或资源泄露。

with 语句 (上下文管理器)

推荐使用 with 语句来处理文件对象，因为它会自动确保文件在代码块执行完毕后（无论是否发生异常）被正确关闭。

读取文件内容 (read, readline, readlines)

假设文件已用读模式打开 (f = open("myfile.txt", "r"))：

  • f.read(size=-1): 读取整个文件内容 (如果 size 未指定或为负)，或读取指定字节数/字符数。
  • f.readline(): 读取文件中的下一行 (包括换行符 \n)。
  • f.readlines(): 读取所有行并返回一个包含每行字符串的列表。
  • 直接迭代文件对象 (推荐逐行读取大文件):

写入文件内容 (write, writelines)

假设文件已用写模式打开 (f = open("output.txt", "w"))：

  • f.write(string): 将字符串写入文件。返回写入的字符数。
  • f.writelines(list_of_strings): 将字符串列表中的所有字符串写入文件 (不会自动添加换行符)。

文件模式 (r, w, a, b, +, etc)

  • 'r': 只读 (默认)。如果文件不存在则抛出 FileNotFoundError。
  • 'w': 只写。如果文件存在则覆盖，不存在则创建。
  • 'a': 追加。如果文件存在则在末尾追加，不存在则创建。
  • 'x': 独占创建。如果文件已存在则抛出 FileExistsError。
  • 'b': 二进制模式 (如 'rb', 'wb')。用于处理非文本文件 (如图像、音频)。
  • '+': 更新模式 (读和写，如 'r+', 'w+', 'a+')。
    • 'r+': 读写。文件必须存在。指针在文件开头。
    • 'w+': 读写。覆盖或创建。指针在文件开头。
    • 'a+': 读写。追加或创建。指针在文件末尾 (写入时)，但在读取时可以移动。

文件指针 (seek, tell)

  • f.tell(): 返回文件指针当前在文件中的位置 (字节偏移量)。
  • f.seek(offset, whence=0): 移动文件指针。
    • offset: 偏移量 (字节)。
    • whence: 参考点。0 (默认) 从文件开头，1 从当前位置，2 从文件末尾。
      (通常用于二进制文件或需要随机访问的文件，文本文件按行处理更常见)。

12. 高级和常用特性

列表推导式 (List Comprehensions)

提供了一种简洁优雅的方式来创建列表。基于现有列表或其他可迭代对象，通过表达式和可选的 for 和 if 子句。
语法: [expression for item in iterable if condition]

字典推导式与集合推导式

类似于列表推导式，但用于创建字典和集合。

  • 字典推导式: {key_expr: value_expr for item in iterable if condition}
  • 集合推导式: {expression for item in iterable if condition}

生成器表达式 (Generator Expressions)

语法与列表推导式类似，但使用圆括号 () 而不是方括号 []。
它们不会一次性创建整个列表，而是返回一个生成器对象 (一种迭代器)。
生成器表达式在处理大量数据时更节省内存，因为它们按需生成值。
语法: (expression for item in iterable if condition)

生成器函数 (Generator Functions with yield)

包含 yield 语句的函数称为生成器函数。调用生成器函数时，它不会立即执行函数体，而是返回一个生成器对象 (迭代器)。
每次对生成器调用 next() (或在 for 循环中迭代) 时，函数会执行到下一个 yield 语句，yield 后面的表达式的值会作为 next() 的返回值。函数的状态（局部变量等）会被保存，下次调用 next() 时从上次离开的地方继续执行。当函数执行完毕或遇到 return 语句 (或没有更多 yield) 时，会抛出 StopIteration 异常。

生成器是实现自定义迭代器的简洁方式。

迭代器 (Iterators) 与可迭代对象 (Iterables)

  • 可迭代对象 (Iterable): 任何可以被迭代的对象，即可以逐个返回其成员的对象。例如列表、元组、字符串、字典、集合、文件对象、生成器。一个对象是可迭代的，如果它实现了 __iter__() 方法 (该方法返回一个迭代器) 或者 __getitem__() 方法 (用于序列)。
  • 迭代器 (Iterator): 代表数据流的对象。它实现了 __iter__() 方法 (返回自身) 和 __next__() 方法 (__next__() 返回序列中的下一个项，如果没有更多项则抛出 StopIteration 异常)。
    for 循环在内部会自动处理迭代协议：首先调用可迭代对象的 __iter__() 获取迭代器，然后重复调用迭代器的 __next__() 获取元素，直到 StopIteration。

装饰器 (Decorators)

一种特殊类型的函数 (或类)，用于修改或增强其他函数或方法的功能，而无需直接修改其源代码。装饰器是高阶函数的一种应用。
语法糖: @decorator_name 放在被装饰函数定义之前。

常用的标准库装饰器包括 @staticmethod, @classmethod, @property，以及 functools.wraps (用于在装饰器中保留被装饰函数的元信息，如名称和文档字符串)。

上下文管理器协议 (__enter__, __exit__)

上下文管理器是支持 with 语句的对象。with 语句确保某些操作在代码块执行之前设置好 (如获取资源)，并在代码块执行之后（无论是否成功）清理干净 (如释放资源)。
要创建上下文管理器，类需要实现：

  • __enter__(self): 在进入 with 块时调用。其返回值赋给 as 子句后的变量 (如果存在)。
  • __exit__(self, exc_type, exc_val, exc_tb): 在退出 with 块时调用。
    • exc_type, exc_val, exc_tb: 如果 with 块中发生异常，这些参数会包含异常的类型、值和追溯信息；否则它们都是 None。
    • 如果 __exit__ 返回 True，则表示异常已被处理，不会向外传播。如果返回 False (或 None，即默认行为)，则异常会继续传播。

类型提示 (Type Hinting - PEP 484)

Python 3.5+ 引入了类型提示，允许开发者为变量、函数参数和返回值指定期望的类型。

  • 目的:
    • 提高代码可读性和可维护性。
    • 帮助静态分析工具 (如 MyPy, Pyright, Pytype) 和 IDE 进行类型检查，在运行前发现潜在的类型错误。
  • 注意: Python 解释器本身在运行时不强制执行类型提示 (除非使用特定库)。它们主要是给开发者和工具看的。

typing 模块提供了许多用于类型提示的辅助类型。

f-字符串 (Formatted String Literals - Python 36)

一种简洁、强大的字符串格式化方式。在字符串字面量前加上 f 或 F，然后在花括号 {} 内直接嵌入表达式。

13. 特定主题简介

处理 JSON 数据 (json 模块)

  • json.dumps(obj, indent=None): 将 Python 对象 (如字典、列表) 序列化为 JSON 格式的字符串。indent 用于美化输出。
  • json.dump(obj, fp): 将 Python 对象序列化为 JSON 并写入文件对象 fp。
  • json.loads(json_string): 将 JSON 格式的字符串反序列化为 Python 对象。
  • json.load(fp): 从文件对象 fp 读取 JSON 数据并反序列化为 Python 对象。

正则表达式 (re 模块)

用于强大的模式匹配和文本处理。

  • re.match(pattern, string): 从字符串开头匹配模式。
  • re.search(pattern, string): 在字符串中搜索第一个匹配项。
  • re.findall(pattern, string): 找到所有匹配项并返回一个列表。
  • re.sub(pattern, repl, string): 替换匹配项。
  • re.split(pattern, string): 根据模式分割字符串。
  • re.compile(pattern): 编译正则表达式模式以提高重复使用的效率。

正则表达式本身是一门复杂的“迷你语言”，需要专门学习其模式语法。

日期与时间 (datetime 模块)

已在标准库部分简要介绍。关键类：

  • datetime.date: 表示日期 (年、月、日)。
  • datetime.time: 表示时间 (时、分、秒、微秒)。
  • datetime.datetime: 表示日期和时间。
  • datetime.timedelta: 表示两个日期或时间之间的差值，或用于日期/时间运算。
  • strftime(): 将 datetime 对象格式化为字符串。
  • strptime(): 将日期字符串解析为 datetime 对象。

异步编程 (async/await - 简介)

Python 3.5+ 引入了 async 和 await 关键字，用于编写异步代码，特别适用于 I/O 密集型任务 (如网络请求、文件操作)，以避免阻塞主线程，提高程序并发性能。

  • async def: 定义一个协程函数 (coroutine function)。
  • await: 在协程函数中调用另一个协程或可等待对象 (awaitable)，暂停当前协程的执行，直到等待的对象完成。
  • 需要一个事件循环 (event loop) 来运行协程，asyncio 模块是 Python 中实现异步编程的核心。

异步编程是一个较大且较复杂的主题，这里仅作概念性介绍。

15. 总结与进阶学习

这份报告涵盖了 Python 的许多常用知识和核心概念。Python 是一门功能强大且用途广泛的语言，其生态系统非常庞大。
进阶学习方向:

  • 深入研究标准库中更具体的模块。
  • 学习特定领域的第三方库，如：
    • Web 开发: Django, Flask, FastAPI
    • 数据科学与机器学习: NumPy, Pandas, Scikit-learn, TensorFlow, PyTorch
    • GUI 开发: Tkinter (内置), PyQt, Kivy
    • 网络编程: socket, requests, asyncio
  • 掌握更高级的 OOP 设计模式。
  • 深入理解并发与并行编程 (多线程、多进程、asyncio)。
  • 学习测试框架 (如 unittest, pytest)。
  • 参与开源项目，阅读优秀代码。

持续实践和构建项目是精通 Python 的最佳途径。
