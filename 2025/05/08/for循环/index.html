for循环

Jackey Zhou2025-05-082025-05-08

1. 遍历集合 (Collections)

这是for循环最常见的用途。它可以遍历实现了IntoIterator trait的任何类型。

a) 遍历值的拷贝 (Consuming the collection)

如果你希望在循环中获得集合中每个元素的所有权（通常用于不再需要原始集合的情况，或者元素类型实现了Copy trait），可以直接遍历集合本身。

如果元素类型实现了 Copy trait（例如 i32, bool 等），则集合不会被消耗，而是每个元素被拷贝。

b) 遍历不可变引用 (Immutable Borrows)

如果你只需要读取集合中的元素，而不需要修改它们，并且希望在循环后继续使用原始集合，可以使用 iter() 方法获取一个迭代器，它会产生元素的不可变引用。

对于大多数集合类型，直接在for循环中使用集合的引用（例如 &v）会自动调用 iter()。

c) 遍历可变引用 (Mutable Borrows)

如果你需要在循环中修改集合中的元素，可以使用 iter_mut() 方法获取一个迭代器，它会产生元素的可变引用。

2. 遍历范围 (Ranges)

Rust的范围表达式 (Range expressions) 实现了 Iterator trait，因此可以直接用于for循环。

a) 半开范围 (start..end)

包含起始值，但不包含结束值。

b) 全闭合范围 (start..=end)

包含起始值和结束值。

3. 结合迭代器适配器 (Iterator Adapters)

Rust的迭代器非常强大，它们有很多“适配器”方法，可以链式调用来转换或过滤迭代器中的元素，然后再用for循环处理。

a) enumerate() - 获取索引和值

enumerate() 方法将迭代器转换为一个产生 (index, value) 元组的迭代器。

b) rev() - 反向遍历

rev() 方法可以反转迭代器的顺序（前提是迭代器支持双向迭代，例如Vec的迭代器）。

c) filter() - 过滤元素

filter() 方法接收一个闭包，只保留闭包返回true的元素。

d) map() - 转换元素

map() 方法接收一个闭包，将迭代器中的每个元素转换为新的元素。

e) zip() - 合并两个迭代器

zip() 方法将两个迭代器合并成一个产生元组的迭代器，当其中一个迭代器耗尽时停止。

f) 其他常用适配器

  • take(n): 获取迭代器前n个元素。
  • skip(n): 跳过迭代器前n个元素。
  • step_by(step): 按指定的步长进行迭代 (用于实现了 ExactSizeIterator 且元素类型为整数的范围)。
  • chain(other_iterator): 连接两个迭代器。
  • collect(): 将迭代器中的元素收集到一个新的集合中（虽然不是for循环直接用，但经常与迭代器处理链一起使用）。

4. break 和 continue

与C++或Java等语言类似，Rust的for循环也支持 break 和 continue 语句。

  • continue: 跳过当前迭代的剩余部分，直接开始下一次迭代。
  • break: 完全退出循环。

5. 标签 (label) 与 break/continue

当存在嵌套循环时，可以使用标签来指定 break 或 continue 应用于哪个循环。

6. for 循环与模式匹配

for 循环的左侧（item 部分）实际上是一个模式 (pattern)。这意味着你可以直接在循环中解构元组、结构体等。

a) 解构元组 (常与 enumerate 或 zip 结合)

b) 解构结构体 (如果结构体实现了 IntoIterator)

通常，你会遍历结构体的某个字段（如果该字段是可迭代的），或者遍历包含结构体的集合。

c) if let 或 while let 风格的过滤 (在循环体内部)

虽然for循环本身不直接支持 if let 形式的过滤，但你可以在循环体内部使用 if let。如果想在迭代器层面进行过滤，通常使用 filter_map 适配器。

这个机制依赖于两个核心 trait:

  • IntoIterator: 任何想要被for循环遍历的类型都需要实现这个 trait。它的 into_iter() 方法返回一个迭代器。
  • Iterator: 这个 trait 定义了迭代器的行为，核心是 next() 方法，它返回一个 Option<Self::Item>。

8. 何时不一定使用 for 循环

虽然for循环非常通用，但在某些情况下，其他结构或方法可能更简洁或更合适：

  • while 循环: 当循环条件不是基于迭代完一个集合时，例如等待某个条件变为真。
  • loop 循环: 用于创建无限循环，或者需要更复杂的 break/continue 逻辑时。
  • for_each() 方法: 如果你只是想对迭代器的每个元素执行一个操作而不关心返回值，for_each() 可能更简洁。
