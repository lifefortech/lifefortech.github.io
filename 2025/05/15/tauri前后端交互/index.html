tauri前后端交互

Jackey Zhou2025-05-152025-05-15

核心交互机制：命令 (Commands) 与事件 (Events)

Tauri 应用中，前端 (Vue.js) 和后端 (Rust) 主要通过两种方式进行通信：

  1. 命令 (Commands): 前端调用后端函数并获取返回值。这是一种请求-响应模式。
    • 用途: 当 Vue 需要 Rust 执行某些操作（如计算、文件操作、数据库查询）并通常期望得到一个结果时使用。
  2. 事件 (Events): 后端主动向前端发送消息，或者前端监听后端发出的特定通知。这是一种发布-订阅模式。
    • 用途: 用于后端状态变更通知、长时间运行任务的进度更新、或者任何后端需要主动通知前端的情况，而无需前端显式请求。

1. 命令 (Commands) - Vue 调用 Rust 函数

这是最直接的交互方式。

步骤分解:

(A) Rust 端：定义和暴露命令

  1. #[tauri::command] 宏:
    • 在 Rust 函数上方添加此宏，Tauri 会自动处理使其能够从 JavaScript 调用的所有底层细节（如序列化/反序列化参数和返回值）。
  2. 函数签名:
    • 函数可以接受基本类型（String, i32, bool, Vec<T>, 自定义结构体等）作为参数。这些参数将从 JavaScript 传递过来。
    • 函数可以返回基本类型、Result<T, E>（用于可失败的操作）或自定义结构体。返回值会被序列化并发送回 JavaScript。
    • 如果函数是异步的 (返回 async fn ... -> Result<T,String> 或类似)，Tauri 也会处理它。
  3. 注册命令 (invoke_handler):
    • 在 src-tauri/src/main.rs 的 tauri::Builder 中，使用 invoke_handler(tauri::generate_handler![rust_function_name1, rust_function_name2]) 来注册所有希望从前端调用的命令。

(B) Vue.js 端：调用命令

  1. 导入 invoke 函数:
    • 从 @tauri-apps/api/tauri 导入 invoke。
  2. 调用 invoke:
    • 第一个参数是 Rust 命令的蛇形命名 (snake_case) 字符串 (例如，Workspace_user_profile)。
    • 第二个参数是一个对象，其键值对应 Rust 函数的参数名和值。如果 Rust 函数没有参数，可以省略此对象或传递一个空对象 {}。
  3. 处理 Promise:
    • invoke 返回一个 Promise。
    • 如果 Rust 函数成功返回 (例如，直接返回值或 Ok(value) from Result)，Promise 会 resolve 并带有该值。
    • 如果 Rust 函数失败 (例如，panic! 或 Err(error_message) from Result)，Promise 会 reject 并带有错误信息。

(C) 数据序列化/反序列化 (幕后)

  • 当 Vue 调用 invoke('cmd', { arg1: value1 }) 时：
    1. { arg1: value1 } 被序列化成 JSON 字符串。
    2. 这个 JSON 和命令名一起发送到 Rust 后端。
    3. Tauri 的 Rust 端接收到请求，找到名为 cmd 的已注册命令。
    4. JSON 参数被反序列化成 Rust 函数 cmd 所期望的参数类型 (例如，String, u32, 或自定义的 UserProfile 结构体，前提是该结构体实现了 serde::Deserialize)。
  • 当 Rust 命令返回时：
    1. Rust 的返回值 (例如 String, UserProfile, Result<UserProfile, String>) 被序列化成 JSON。如果是 Result::Err, 错误部分被序列化。
    2. 这个 JSON 被发送回 Vue 前端。
    3. @tauri-apps/api 的 invoke 函数接收到 JSON，将其反序列化成 JavaScript 对象或值，并 resolve (或 reject) Promise。

关键点:

  • 类型匹配: JavaScript 传递的参数类型应能被 Rust 函数的参数类型正确反序列化。自定义结构体在 Rust 端需要 derive(serde::Deserialize)，如果作为返回值则需要 derive(serde::Serialize)。
  • 错误处理: Rust 的 Result<T, E> 非常适合可失败的操作。Ok(value) 会使 Promise resolve，Err(error) 会使 Promise reject。这使得前端可以清晰地处理成功和失败的情况。

2. 事件 (Events) - Rust 主动通知 Vue

事件允许 Rust 后端在任何时候向一个或多个前端窗口发送数据，而无需前端发起请求。

步骤分解:

(A) Rust 端：发送事件

  1. 获取 AppHandle 或 Window:
    • 在 setup 钩子中，可以通过 app.handle() 获取 AppHandle。
    • 在命令中，可以接收 tauri::AppHandle 或 tauri::Window 作为参数。
    • AppHandle 用于向所有窗口发送事件 (app_handle.emit_all("event-name", payload)).
    • Window 用于向特定窗口发送事件 (window.emit("event-name", payload)).
  2. emit_all 或 emit 方法:
    • 第一个参数是事件名称 (一个字符串，例如 "backend-update")。
    • 第二个参数 (可选) 是负载 (payload)，可以是任何可序列化为 JSON 的数据 (需要 serde::Serialize)。
  3. 后台任务中发送事件: 如果您有在后台线程中运行的任务（例如，通过 std::thread::spawn 或 tokio::spawn），您需要将 AppHandle 克隆并 move 到线程闭包中，以便从该线程发送事件。

(B) Vue.js 端：监听事件

  1. 导入 listen 函数:
    • 从 @tauri-apps/api/event 导入 listen。
  2. 调用 listen:
    • 第一个参数是要监听的事件名称 (必须与 Rust 中 emit_all 或 emit 使用的名称完全一致)。
    • 第二个参数是一个回调函数 (event) => { ... }。当 Rust 发送匹配名称的事件时，此回调会被执行。event.payload 包含了从 Rust 发送过来的数据。
  3. 返回 UnlistenFn:
    • listen 函数返回一个 Promise，该 Promise resolve 为一个 UnlistenFn 函数。调用这个 UnlistenFn 可以取消对该事件的监听。这在组件卸载时非常重要，以防止内存泄漏或不必要的处理。
  4. 类型化 Payload:
    • 可以为 listen 提供一个泛型参数来指定 event.payload 的类型，例如 listen<MyPayloadType>(...)。

(C) 作用域与目标

  • app_handle.emit_all("event-name", payload): 将事件发送到所有打开的、并且正在监听该事件名称的 Tauri 窗口。
  • window.emit("event-name", payload): 将事件仅发送到调用此方法的特定 window 实例，前提是该窗口的前端代码正在监听此事件。
    • 要获取特定窗口，您可以在 Rust 中使用 app_handle.get_window("window_label")，其中 "window_label" 是在 tauri.conf.json 中或创建窗口时指定的唯一标签。

3. 安全性: allowlist 的角色

所有这些交互都受到 tauri.conf.json 中 allowlist 的严格控制。

  • 调用命令: 默认情况下，前端不能调用任何 Rust 命令。您不需要在 allowlist 中列出每个命令的名称。invoke_handler 的注册本身就隐式地允许了这些命令的调用，但 Tauri 的核心安全模型（如 IPC 消息校验）仍然适用。
  • 发送/接收事件:
    • 要使 app_handle.emit_all 或 window.emit 工作，前端的 allowlist 中需要允许 event API，特别是 event.listen 的能力。

总结：Vue 与 Rust 的双向数据流

  • Vue -> Rust (请求数据或执行动作):

    1. Vue 组件使用 @tauri-apps/api/tauri 的 invoke('rust_command_name', { args })。
    2. 数据通过 JSON 序列化后发送给 Rust。
    3. Rust 端对应的 #[tauri::command] 函数被执行，接收反序列化后的参数。
    4. Rust 函数处理逻辑，返回结果 (或 Result<T,E>)。
    5. 结果通过 JSON 序列化后发送回 Vue。
    6. Vue 端 invoke 的 Promise resolve (或 reject)，Vue 更新 UI。

  • Rust -> Vue (主动推送更新或通知):

    1. Rust 端（例如在后台线程、setup 钩子或某个命令执行完毕后）获取 AppHandle 或 Window 实例。
    2. 调用 app_handle.emit_all("event_name", payload) 或 window.emit("event_name", payload)。
    3. payload 通过 JSON 序列化后发送给前端。
    4. Vue 组件中，通过 @tauri-apps/api/event 的 listen('event_name', callback) 注册的监听器被触发。
    5. 回调函数中的 event.payload 包含反序列化后的数据，Vue 更新 UI。

这种结合了请求-响应和发布-订阅的模式，为构建功能丰富的 Tauri + Vue 桌面应用提供了强大而灵活的通信基础。
